# 🧪 第一课：推导出你的 Hello World

::: danger 现状警示
根据我的观察，很多大学生上完一学期课，哪怕考完试都不能独立写完打印 Hello World。这不怪学生，怪的是那种“死记硬背”的教法。今天我们不背代码，我们来“推”代码。
:::

## 🎯 1. 明确目的：我要说话

我们的目的很简单：让计算机在屏幕上输出 `hello world`。

经过学习发现，C 语言中输出函数是 `printf`。但 `printf` 不属于 C 语言的“本能”，它属于外部库。

> **学长笔记：做饭与吃饭 (理解外部库)**
> 
> - **吃饭（内置本能）**：你天生就会。类比 C 语言里最基础的 `putchar`、`sizeof` 等。
> - **做饭（外部技能）**：你得先“学习”并“导入”做饭的技能包。
> - **`stdio.h`**：这就是那个“标准输入输出技能包”。没有 `#include <stdio.h>`，计算机就不认识 `printf`。

---

## 🛠️ 2. 语法装裱：让指令生效

如果我们直接写 `printf(hello world)`，计算机会看不懂。我们需要按照 C 语言的“赛博规范”进行装潢：

1. **#号指令**：`include` 前面要加 `#`。
2. **语法规范**：库引用可以用 `<>` 或 `''`。
3. **引号包裹**：输出的文本内容必须有引号 `""`。
4. **句末分号**：每句指令末尾要加 `;` 表示写完了。

此时，我们的代码进化到了这一步：

```c
#include <stdio.h>
printf("hello world");
```

---

## 🏗️ 3. 寻找入口：为什么要有 main(){}？

计算机执行指令时需要一个“大门”。没有这个门，计算机不知道从哪开始跑，也不知道哪些指令该执行。

- **`main` 函数**：主函数，计算机看到它就知道这部分要求被执行。
- **大括号 `{}`**：代表门里的东西都属于这个函数内部。
- **缩进**：展示这种从属关系。

```c
#include <stdio.h>
main() {
	printf("hello world");
}
```

---

## ⚖️ 4. 反馈结果：谈谈 int 和 return

计算机执行完任务，得有个反馈。我们约定：**0 代表成功，非 0 代表失败**。

1. **返回值**：既然要返回 0，0 是个整数，我们就需要在 `main` 前面标明类型是 `int`（整数类型）。
2. **收尾**：在最后加上 `return 0;`。

### 🧠 揭秘：打破“必须这么写”的迷信
其实代码写成什么样子完全看自己心意，只是行业规范建议用第一种。为了让你理解本质，看看这些“任性”但逻辑自洽的写法：

| 写法 | 代码示例 | 说明 |
| :--- | :--- | :--- |
| **标准写法** | `int main() { ... return 0; }` | **推荐**。最规范，返回整数 0。 |
| **无返回值** | `void main() { ... }` | 告诉计算机：跑完就行，不用给我回话。 |
| **精度类型** | `double main() { ... return 0.99; }` | 返回一个带小数的结果，完全可行。 |
| **任性返回** | `int main() { ... return 9999; }` | 只要你愿意，返回 9999 也可以。 |

---

## 🚀 最终成品：你亲手推导的第一个程序

至此，我们的第一个程序正式诞生了。不再是死记硬背，而是逻辑使然。

```c
#include <stdio.h>

int main() {
	printf("hello world");
	return 0;
}
```
